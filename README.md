# extend-db-hybrid

Local ExtendDB setup and a small Streamlit app for inserting/querying DynamoDB-style items.

## Local ExtendDB Quickstart

### 1) Start the local stack

```bash
docker compose up -d --build
```

This starts:

- `postgres` on `localhost:5432`
- `extenddb` on `localhost:6688`
- `pgadmin` on `localhost:8080`

### 2) Export the TLS certificate

```bash
./scripts/export-cert.sh
source certs/extenddb-aws-env.sh
```

This copies the ExtendDB certificate from the container and sets:

- `AWS_ACCESS_KEY_ID`
- `AWS_SECRET_ACCESS_KEY`
- `AWS_REGION`
- `AWS_CA_BUNDLE`

### 3) Verify the endpoint

```bash
aws dynamodb list-tables \
	--endpoint-url https://127.0.0.1:6688 \
	--region us-east-1
```

You should see an empty `TableNames` list the first time.

## Create a Sample Table

Use the local endpoint and the exported credentials to create a table. Example:

```bash
aws dynamodb create-table \
	--endpoint-url https://127.0.0.1:6688 \
	--region us-east-1 \
	--table-name events \
	--attribute-definitions AttributeName=id,AttributeType=S AttributeName=timestamp,AttributeType=S \
	--key-schema AttributeName=id,KeyType=HASH AttributeName=timestamp,KeyType=RANGE \
	--billing-mode PAY_PER_REQUEST
```

Insert a sample item:

```bash
aws dynamodb put-item \
	--endpoint-url https://127.0.0.1:6688 \
	--region us-east-1 \
	--table-name events \
	--item '{"id":{"S":"user-123"},"timestamp":{"S":"2026-05-31T12:00:00+00:00"}}'
```

## Run the Streamlit App

Install dependencies:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Run the app:

```bash
streamlit run app.py
```

In the sidebar, set:

- AWS Region: `us-east-1`
- Table Name: `events`
- DynamoDB Endpoint: `https://127.0.0.1:6688` (or your local endpoint)

## Notes

- For local ExtendDB access, use the AWS credentials generated by the container, not your personal AWS profile.
- If you re-create the stack with `docker compose down -v`, re-run `./scripts/export-cert.sh` and source the env file again.
